# CLAUDE.md 越长 Agent 越不听 — 5个常见坑

作者: Ali厂长 | 粉丝1170 获赞6664 | 发布时间: 2026-05-08 | 时长: 6:01
数据: 235赞 3评 309收藏 42分享

## 章节要点
CLAUDE.md和AGENTS.md是context注入文件，用于与agent对话。常见的五个问题包括规则过多、只禁不导、大段文档引用、命令不具体和临时状态永久化。

### 坑1: 规则过多 (00:37)
规则过多会导致规则被稀释，建议保持在200行以内，每条规则都问一句是否可以删除，纯风格规则交给工具处理。

### 坑2: 只禁不导 (01:42)
只禁不导会让agent在解空间里随机游走，建议每条规则禁止用什么都顺手补一句推荐用什么。

### 坑3: @路径引用大段文档 (02:04)
用@路径引用大段文档会导致全量加载，建议在引用前面写清楚什么时候去读。

### 坑4: 命令不具体 (02:37)
命令不具体或漏掉会导致agent猜测错误，建议写清楚单文件测试命令、patch命令和部署命令。

### 坑5: 临时状态永久化 (03:18)
临时状态永久化会导致过期事实，建议将临时状态放到独立的临时文件里，按需加载。

### 自检方法 (04:00)
通过 tinkleberry test 和直接提问 test 进行自检。

### 结语 (05:34)
CLAUDE.md和AGENTS.md是软约束，不能保证什么，真正的硬护栏要靠 Hooks/Ceilings。

## 完整描述
我的 CLAUDE.md 从 30 行写到 1000 行，Agent 反而越来越不听话。
CLAUDE.md / AGENTS.md 不是文档，是 context 注入文件。你写的每一行，都会和正在改的代码、对话历史、工具输出一起塞进同一个 context window 里抢注意力。它越长，能抢到的注意力越少。
