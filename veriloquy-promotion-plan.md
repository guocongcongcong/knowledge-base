# Veriloquy（以言求真）推广方案

> 大白话 · 可执行 · 不装逼 · 八方向全覆盖
>
> 写给谁看的：你，一个写网文的个人开发者，业余时间折腾了套 AI 家教系统，想让更多人用上它。
>
> 核心原则：**不投一分钱，全靠内容和真诚。**

---

## 方向一：GitHub 开源推广

### 推给谁

**两类人**：一是自己也在捣鼓 AI 项目的开发者（技术上看得懂你在干什么），二是想用 AI 学东西但觉得 ChatGPT 太"硬"的学习者（虽然不写代码，但愿意上 GitHub 逛逛）。

### 具体怎么做

**第一步：把 README 做得让人三秒看懂**

现在的 README 有 661 行，内容很全。但问题就在这里——太长。一个路人进来，滑三秒滑不到底，直接关掉。

你需要：

1. **首页折叠**：README 最上面保留 15 行以内的"一句话版"——"这是一个用 AI 四夫子追着你问到底的学习系统，像真人老师一样跟你对话。" 够短、够具体、够有画面感。
2. **加 GIF 演示**：录一个 30 秒的使用过程——打开 → 选一个夫子 → 开始问对。放 README 第一屏。工具用 `asciinema`（命令行版）或 `Kap`（macOS 录屏，开源免费）。
3. **Star 徽章 + CI 徽章**：GitHub Action 跑一遍学籍簿校验，放个绿 badge。看着专业。
4. **中文优先，英文备份**：README 首语言用中文。另外建一个 `README.en.md`，给人感觉"我们有国际化 sense"。

**第二步：用 Issue 当论坛**

GitHub Issues 不只是报 bug 的地方。你在上面：
- 开一个 **"你用过四个夫子了吗？来分享你最喜欢谁"** 的讨论贴
- 开一个 **"你希望 Veriloquy 教什么内容"** 的征集贴
- 每周开一个 **"本周修路进度 🛤️"** 的日志贴，让大家看到你还在干活

这不是运营，这是**让人看到你有在持续做事**。

**第三步：主动去别人的项目露脸**

找那些跟 Veriloquy 气质相关的开源项目：
- AI 教育类（`llm-examples`, `open-interpreter` 等）
- 学习管理类（`anki`, `remnote` 的相关讨论）
- 中文 Prompt 工程类

不是去发广告。而是：
- 在别人 Issue 里认真回答问题（比如有人问"LLM 怎么教数学"，你可以说"我试过一种追问式...")
- 在 Discussion 里分享你的经验
- 别人 star 了你的项目，你也看看他项目的 README，点个星星，留句有意义的话

**第四步：v1.0 发版时搞"Release 仪式"**

GitHub Release 不能只是 changelog。每次发版写一段"叙事"：
- v0.1："鹿鸣夫子醒了——第一个能教 AI Agent 的 AI 老师"
- v0.2："素问来了——数学终于不用硬背了"
- v1.0："四位夫子到齐，孔夫子校训写好"

这种命名方式本身就让人想点进来看。

### 需要准备什么

- 一个简短的演示 GIF（不超过 30 秒，重点展示"追问"这个过程）
- 两三个友好的 Issue/ Discussion 模板（用来引导大家说话）
- 一条发布清单（发版前 Check：Changelog 写了没？Demo GIF 新的没？）

### 预计效果

- 前 3 个月：50-200 个 Star（正常个人项目冷启动数据）
- 6-12 个月：500-2000 个 Star（如果持续输出内容）
- 会有 3-10 个人提 PR/Issue，这些人是你的第一批"共建者"

### ⚠ 注意坑

- **不要刷 Star**。一眼假，Github 会检测，社区也会反感。
- **不要整天看 Star 数**。开源项目前半年没人关注是正常的，你唯一要做的就是"别停"。
- 不要在 Reddit/HN 上发帖求 Star——没有上下文的自推会被骂。

## 方向二：知乎/少数派 文章

### 推给谁

**知乎**：学编程学 AI 但觉得现有教程太枯燥的人（大学生、转行程序员、想做 AI 应用的产品经理）。
**少数派**：工具控、效率控、喜欢新鲜玩意的数码/学习爱好者。
**两个平台本质上都是"长文读者"——恰恰适合你的文笔。**

### 具体怎么做

**先写好三篇核心文章**（不要一次发完，间隔 1-2 周）：

**文章一：《我用四个"不讲课的老师"学会了一门新技能》**

- 开头：你有没有碰到过这种情况——上课/看书时觉得"懂了"，合上就忘？Veriloquy 要解决的就是这件事。
- 中间：用自己的初次使用体验来写。比如打开系统 → 选了"素问"夫子 → 她问了我第一个问题 → 我回答了 → 她接着问 → 半小时后我发现，我不知不觉理解了之前卡了很久的概念。
- 关键：用**叙事**而不是**功能列表**。不是说"我们有四位夫子"，而是写"素问是怎么一步步把我问明白的"。
- 结尾：说这个项目是开源的，可以自己部署，也欢迎来体验。放 GitHub 链接。

**文章二：《为什么孔子门下的学生那么牛？我用 AI 复刻了这种教学方式》**

- 开头：讲讲《论语》里的教学场景——孔子很少长篇大论，更多是"因材施问"。
- 中间：引出 Veriloquy 的核心理念——"学问生于问对之间"。介绍四位夫子的设计思路（不要全部列，重点讲一个你最喜欢的）。
- 收尾：拉回到现实——我们现在都用 AI 工具了，但能不能用它来"真学会"东西，而不是让它替我们干活？

**文章三：《我业余写了半年，这是我的 AI 家教系统——Veriloquy 完全指南》**

- 这其实就是你个人故事的完整版：怎么想到要做的 → 遇到了什么困难 → 现在的样子 → 未来的方向。
- 这种文章在少数派上特别吃香，因为少数派的读者就是喜欢看"个人做产品"的故事。
- 重点不是技术细节，是"一个人做了这个东西"这件事本身。

**以上三篇，两投知乎，一投少数派。可以交叉发表（比如知乎发三篇，少数派发一篇精华版）。**

**回答相关问题（长期动作）**

- 在知乎搜"怎么学XXX最有效""AI 能用来学习吗""有哪些好用的学习工具"——在下面回答，顺带提 Veriloquy。不是打广告，是真心分享。格式："我试过很多方法，最近发现一种有意思的方式..."
- 在别人的回答下评论：写 100 字以上的有用评论，比你自己发一篇文章的有效传播率还高。

**善用专栏**

如果回答积累够多了，开一个知乎专栏，叫"以言求真"或者"自学不孤独"。把所有 Veriloquy 相关文章放进去，形成一个系列。

### 需要准备什么

- 三篇文章的选题方向确认（上面已经给了大致思路，你需要写出全文）
- 几个使用截图（显示对话界面、四个夫子出场、修路进度等）
- 一个简单的概念图（四个夫子的头像 + 一句话介绍）

### 预计效果

- 每篇文章：500-5000 阅读（取决于文章质量和运气）
- 如果能上知乎"编辑推荐"或少数派首页：2-10万阅读
- 每篇会带来几十到几百个 GitHub 访问

### ⚠ 注意坑

- **不要在文章里写"求 Star"**。文章末尾自然挂链接就够了。逼求的效果反而是反的。
- 不要写得太 technical。90% 的读者不在乎"背后用了什么 LLM 技术"，只在乎"这个东西跟我有什么关系"。
- 不要在评论区跟喷子吵架。你不可能说服所有知乎用户。遇到抬杠的，要么不回复，要么回一句"你说的有道理，但我的场景不太一样"。

## 方向三：B站/视频号 视频内容

### 推给谁

**B站**：学习区/知识区的年轻人（大量大学生、自学者、备考党）。
**视频号**：微信生态里的成年人（30+，想学东西但时间碎片化）。

### 具体怎么做

**B站为主，视频号同步搬运。**

你的优势：你会写东西。**B站知识区的爆款视频，脚本 > 剪辑 > 口条 > 脸。** 你的文笔可以转化为高质量的脚本。

**第一类视频：实操演示（最容易）**

标题风格：**"我用 AI 四个夫子学了一整天，我人麻了"**
- 开头：10 秒内上钩。"你见过这样的 AI 老师吗？她不给你答案，反而一直问你问到你崩溃。"
- 中间：展示真实的对话过程。打开终端/Web → 选一个夫子 → 开始对谈。把"追问"这个过程完整展现出来。
- 过程里穿插你的真实反应（"啊，被她问住了""等一下，我好像明白了"）。
- 结尾：总结这个体验。放 GitHub 地址。
- 时长：3-5 分钟。

**第二类视频：理念阐释（适合长视频）**

标题风格：**"为什么学过的知识总是忘？两千年前孔子就知道答案了"**

- 用 Veriloquy 背后的理念（修路八阶、追问式教学）做主题，但不是讲产品功能，而是讲"学习方法论"。
- 这种视频即使不用 Veriloquy 也能成立——大 V 会转发，普通观众会收藏。
- 只要在视频末尾说一句"我照这个理念做了个开源工具，叫 Veriloquy"，就能自然引导过去。

**第三类视频：四人联播（整活向）**

标题风格：**"让四个不同性格的 AI 老师教同一道题，场面失控了"**

- 用一个知识点（比如"什么是递归"），分别让四位夫子来教。
- 对比四位夫子的风格：鹿鸣会追着你问代码 → 素问会耐心引导 → 断章会直戳要害 → 观澜会哲学追问。
- 这种视频有"综艺感"，看完的人会自发讨论"你最喜欢哪个夫子"。
- 能带动用户自发在评论区猜哪个夫子最厉害、谁最温柔——评论区本身就是传播。

**制作标准：**

- 不需要露脸（至少前期不需要）
- 用 OBS 录屏 + 自己的声音旁白
- 剪辑用 DaVinci Resolve（免费）或剪映（上手快）
- 每支视频预留 2-3 天写脚本 + 1 天录制 + 1-2 天剪辑
- 前三个月，每月至少出 1-2 个视频

**视频号策略：**

- B站视频剪辑成 1-3 分钟的短视频版，发视频号
- 配文要更直接、更"微信体"：字数少，句子断，重点突出
- 视频号的推荐机制跟社交关系链相关，发完后转朋友圈、转群聊

### 需要准备什么

- 一个好听的麦克风（业余不够用手机自带也行，但声音要清晰）
- 写脚本的能力（你有，这个最强）
- OBS 录屏 + 简单剪映熟练度
- 一两个经过测试的、不会翻车的演示素材（提前跑一遍确保所有对话流畅）

### 预计效果

- B站知识区第一个视频：500-3000 播放（冷启动正常）
- 如果有一个视频达到 1 万+ 播放，后面的就会被推荐算法多推
- 视频号：200-2000 播放（主要靠朋友圈转发）
- 视频第 1 分钟完播率 > 30% 就算及格，> 50% 就是爆款候选

### ⚠ 注意坑

- **不要在视频里放一个超长的命令安装步骤**。观众会立刻关掉。把零代码版体验放前面，部署放后面或放评论区置顶。
- B站的用户非常敏感"恰饭感"。真诚分享和打广告的区别，B站观众一眼就能看出来——你是在分享一个有趣的东西，不是在卖课。
- 不要想着"一条视频爆了怎么办"——大多数人第一条视频连 1000 播放都没有。坚持 6 个月才会有起色。
- 视频号的流量来得快去得也快，不要死磕。

## 方向四：高校/教育社群 学术合作

### 推给谁

- **高校教师**：特别是教"教育技术""学习科学""AI 应用"这几类的老师——他们一直在找能让自己的课"有点新意"的东西。
- **教育类社团/实验室**：北师大的学习科学实验室、华东师大的教育技术系、各大高校的"AI 教育"社团。
- **中小学信息老师**（如果你愿意把 Veriloquy 改成适合青少年的版本）：比较难，先放一放。

### 具体怎么做

**不要一上来就说"我是来做合作的"。**

**第一步：在学术社群里"做人"**

- 关注微信公众号：学习科学、教育技术、AI 教育等方向（推荐 "教育技术前沿""学习科学""AI 虫子" 等）。
- 加入相关微信群/QQ群（很多都会有"教学交流群"）。
- 在里面当个"好人"——不是去推广 Veriloquy，而是：别人遇到技术问题你帮忙解决；别人讨论教学方法你真诚分享自己的思路；别人推荐工具你试用后给真实反馈。
- 多做一个月，你的名字就混熟了。

**第二步：自然导入**

当你已经成了群里被认识的人之后，可以在讨论中提一句："我之前试过一种追问式的 AI 教学方式...做了一个小工具，还在打磨阶段..."

这时候如果有人感兴趣，不要急着推，可以说"还在改，等差不多稳定了私聊发你一个体验方式"。

**第三步：定向邀约试用**

对于真正感兴趣的老师：
- 请他们"用用看，给点意见"——而不是"请用我的产品"
- 告诉他们"这是开源的，你可以自己改着用"
- 收集他们的使用反馈，作为改进方向
- 如果效果好，他们会在课堂上用，然后帮你传播到其他学校

**第四步：考虑写一篇学术短文**

如果你有余力，可以写一篇 3000-5000 字的短文，讲"追问式 AI 教学的实践探索"，投到比较开放的在线教育论坛或者微信公众号。不需要正式的学术论文格式，但要有理有据（你可以用 Veriloquy 的实际对话记录做素材）。

### 需要准备什么

- 你的"学术互动时间"：每周至少花 1-2 小时在相关社群中
- 一个简单的"教师使用指南"：告诉老师怎么在课堂上用 Veriloquy（比如，给学生布置一个任务：选一个夫子学一个概念，然后写心得）
- 收集使用反馈的模板：不必复杂，几个问题就行——"哪里好用""哪里卡住了""你希望加什么"

### 预计效果

- 前 3 个月：可能只有 1-3 个老师真正用起来
- 半年到一年：如果效果好，老师会有动机在其他场合推荐
- 长期来看：高校合作带来的"权威背书"效应远超普通用户传播

### ⚠ 注意坑

- **不要群发"老师您好"的私信**。99% 会被无视或拉黑。
- 不要在学术群里发 GitHub 链接。在学术社群里，你要显得"有深度"而不是"在卖东西"。
- 高校的决策周期特别长。一个老师从"感兴趣"到"在课堂上用"可能需要两三个月。要有耐心。
- 不要期望高校给你钱。高校合作更多是"名"和"反馈"。

## 方向五：AI 比赛/Hackathon

### 推给谁

- **评委和参赛者**：不是泛泛的"AI 比赛"，而是教育科技类、AI 应用类、开源创新类。
- 你未来可能的同路人——Hackathon 是认识靠谱人的最佳场合。

### 具体怎么做

**选对比赛，比"做出完美 Demo"更重要。**

适合你的比赛类型：
- **教育科技 Hackathon**（每年都会有教育局、企业、高校合办的）
- **AI 应用创新大赛**（很多科技公司联合举办的）
- **开源之夏**（如果你有时间在暑期做）
- **各类"AI + 教育"的创业/创新赛**（有很多不需要注册公司）

**参赛前准备：**

1. **准备一个"Demo 视频"**：3 分钟以内，展示闭环。核心展示"追问式教学"这一个场景就够了——选一个知识点，让一个夫子从头问到尾。让人看完就明白"这跟别的 AI 学习工具不一样"。
2. **准备一个"一句话梗概"**：30 秒内说出"我在做什么"——"我是一个基于孔子教学理念的 AI 家教系统，四位数字夫子用全程追问的方式帮你真正学会一个知识点，而不是给你答案。"
3. **准备一份"技术实现说明"**：最多一页纸，讲清楚用了什么技术、怎么做到的、学到了什么。
4. **准备一个"它能长成什么样"的愿景**：不是商业计划，而是"如果给更多人用，它能改变什么"。

**比赛中的策略：**

- 展示的时候**演示 > 讲稿 > 文档**。一个流畅的 3 分钟演示胜过 30 页 PPT。
- 讲"为什么做"而不是"做了什么"。评委一天听了几十个项目，能记住的都是那些"有故事"的。
- 如果入围决赛，准备好被拷问。常见问题："你跟 ChatGPT 让用户自己提问有什么区别？""你怎么确保用户不是在跟 AI 聊天？""你打算怎么让更多人来用？" 想清楚再回答。

**比赛后：**

- 不管获不获奖，加微信。你在比赛现场认识的 20 个人，可能比 200 个线上粉丝更有价值。
- 赛后写一篇复盘发到各个平台（知乎、公众号等）。标题："我拿（某比赛）的 AI 项目去参加（某 Hackathon），学到了什么"。

### 需要准备什么

- 一台能流畅运行 Veriloquy 的电脑（演示用的）
- 一个"离线版"demo 环境（比赛现场网络可能不稳定）
- 3 张能讲清核心卖点的图（夫子四合一、修路八阶图、一个对话截图）
- 你的"故事线"——用大白话讲清楚一个完整的故事
- 各种插头转接器（HDMI转USB-C等）

### 预计效果

- 如果拿奖：一个月内会有 200-1000 个新访客来 GitHub
- 如果不拿奖：你会结识很多做类似事情的人，互相交流
- 肯定会有的收获：对 Veriloquy 的 debug 能力大幅提升（比赛逼着你把细节打磨好）

### ⚠ 注意坑

- **不要做"完美主义拖延"**。"等我把 V8 做完再去参加比赛"——永远不会有做完的那一天。现在这个 MVP 就够了。
- 比赛现场的设备总是掉链子。提前 2 小时到场调试。
- 不要超时。如果给你 5 分钟，练到能在 4 分 30 秒内讲完。超时就是扣分项。
- 不要假装自己是创业公司。诚实地说"这是一个个人项目"反而更让人尊重。

## 方向六：技术社区分享（V2EX/掘金等）

### 推给谁

**V2EX**：程序员、创业者、科技爱好者——比较杂，但程序员多。
**掘金**：年轻前端/全栈开发者，对"AI 应用"类内容接受度最高。

这两个社区是"开发者社区"，所以分享的风格要**偏技术但不要太硬**。

### 具体怎么做

**V2EX 策略：**

V2EX 有一个特点——只要你发的内容真的有意思、真的有用，大家会自发点赞收藏。反过来，如果你发的内容像广告，就会被踩。

适合 V2EX 的帖子类型：

**帖子一："我用 LLM 做了一个会追问的 AI 老师——效果比想象中好"**

- 内容：技术实现的角度。用了什么模型、怎么设计 Prompt 实现"追问"机制、四夫子人设是如何通过角色设定调出来的。
- 不要写代码，写思路：怎么控制 LLM 不直接给答案 → 怎么判断学生是真的理解了还是在糊弄 → 怎么设计"修路"的时序。
- 重点展示你知道自己在做什么技术，而不是"看我的项目很牛"。
- 发在 `/go/ai` 或 `/go/creativity` 节点。

**帖子二："分享一下我做的零代码 AI 家教系统的学籍簿——怎么追踪学习进度"**

- 讨论学籍簿的设计思路：怎么用 Markdown 文件记录学习状态、怎么判断复习时机、怎么让"进度可视化"。
- 这种技术细节帖在 V2EX 上很受欢迎——别人看完会有"学到了"的感觉。

**帖子三（后续）："我开源六个月了，聊聊一个个人项目的真实数据"**

- 六个月后发一篇复盘帖：GitHub Star 增长曲线、最受欢迎的模块、最难的地方、用的人反馈了什么。
- 这种帖子在 V2EX 上自带热度——大家都喜欢看"真实的故事"。

**掘金策略：**

掘金的用户偏年轻，更喜欢"手把手教程"类的文章。

**文章一："10 分钟搭建你的 AI 私人家教系统（零代码版）"**

- 教程向：show 怎么 clone 项目 → 配 API Key → 启动 → 开始学习。
- 不需要写太多原理，重点是"跟着我就能跑起来"。
- 掘金喜欢步骤清晰、图文并茂、代码块漂亮的文章。

**文章二："我在 AI 家教系统里实现了四种教学风格——实现对比"**

- 技术分析：四位夫子的 Prompt 设计有哪些不同、如何控制输出风格。
- 这种文章在掘金上会吸引"Prompt 工程"这个话题的读者。

**发帖频率：**

一个月 1-2 篇就够了。不要太频繁，否则会被视为"营销号"。

**别忘了在帖子末尾自然带链接：**

> "我把这个项目开源了，代码在 GitHub：链接。欢迎 Star、Fork、提 Issue。反正一个人做挺孤独的，有人一起玩就开心了。"

### 需要准备什么

- 至少 2-3 篇的素材（写好后可以复用/改编发到不同平台）
- 每个平台熟悉一遍：注册账号、看明白发帖规则、了解社区氛围
- 记住一个原则：**先贡献，再分享**。在你发自己的帖子之前，先去别人帖子里认真回复 10 次

### 预计效果

- V2EX 一篇热帖：5000-20000 阅读，100-300 个收藏
- 掘金一篇好文章：3000-10000 阅读，100-500 个点赞/收藏
- 转化到 GitHub：阅读量的 2%-5%

### ⚠ 注意坑

- **别在 V2EX 发纯推广帖**。会把账号权重打低。
- 不要在文章里放二维码——V2EX 不允许，掘金也不太合适。放链接就够了。
- 不要刷赞和留言。一眼假，社区会封号。
- 掘金上"AI 教程"类文章特别卷，你的差异点一定要在前 3 段就亮出来。

## 方向七：微信社群裂变/运营

### 推给谁

愿意认真学点东西、对传统教育方式有点不满、觉得"AI 可以帮忙但不知道怎么用"的人。

### 具体怎么做

**不急着建群。先个人号，后社群。**

**第一步：打造个人号形象（1-2 个月）**

你的朋友圈可以变成"Veriloquy 开发日志"：
- 今天修了哪个夫子的人设
- 今天被某个夫子的回答惊到了
- 今天发现了一个之前没注意到的 bug
- 今天收到第一个用户反馈说"这个有用"

这些东西发在朋友圈，不用刻意推广。有意的人自然会好奇，留言问"这是什么"。

**第二步：拉一个种子群（20-50 人）**

建群动机不要是"来体验我的产品"，而应该是一个具体的目标：

> "我想找 20 个朋友一起用这个 AI 工具学点东西，我提供工具，你们提供真实反馈。"

比"快来用我的产品"好听一百倍。

**群规就三条：**
1. 不聊与学习无关的事（别让群水掉）
2. 每个月至少学完 1-2 个概念，在群里分享心得
3. 遇到问题在群里问，大家互相帮助

**第三步：群内营造"共学"氛围**

每天或每两天发一个"今日题"——一个知识点，让大家去跟任意一位夫子对谈，然后截图发群里。
你作为群主的角色是：
- 回应每个分享的人（哪怕只是点个赞）
- 回答技术问题（"这个夫子为什么回答得这么长？""怎么清理对话记录？"）
- 偶尔抛出话题（"你们觉得哪个夫子最严？"）

**第四步：自然裂变（不靠刷屏海报）**

当群里有了 5-10 个活跃用户后，可以提一句：
> "如果你身边有朋友也想试试这个学习方式，可以拉进来。群规照旧就好。"

不用"转发海报截图入群"那一套——那些适合大众产品，不适合一个还在打磨阶段的 AI 系统。

**第五步：如果你有余力，试试"陪伴式学习"**

Veriloquy 本身是 AI 在教，但人其实是需要"陪伴感"的。你可以每周花 1-2 小时，在群里做"活人互动"：
- 选一个知识点，在群里直播自己跟夫子对谈
- 或者让大家投票选一个知识点，你跑一遍给所有人看
- 或者收集大家这周的"学习问题"，集中答疑

这不是在"运营"，这是**你在展示你也在学习**。一个"带头学习的人"比任何运营手段都有效。

### 需要准备什么

- 一个专门用于 Veriloquy 的微信号（如果你愿意用主号也行，但消息会很多）
- 群规：简单、明确、不烦人
- 每日/每周的"带学内容"——你已经有一整个 V1-V8 体系，随便挑一个教材就能聊
- 耐心：社群不可能一夜之间热闹起来。前两个月可能只有三五个人偶尔说话

### 预计效果

- 第 1 个月：15-20 人入群，其中 3-5 人活跃
- 第 3 个月：40-60 人，8-15 人活跃
- 半年后：稳定 80-150 人的学习社群
- 核心效果不是"多少人进来了"，而是"有十几个人是真的在用"——这已经很好了

### ⚠ 注意坑

- **不要搞"打卡返现""分享得奖励"。** 你掏不起，而且会吸引来薅羊毛的而不是学习者。
- 不要在群里打太多广告。这个群的核心是"一起学习"，不是"Veriloquy 用户群"。
- 不要因为活跃度低而焦虑。一个群有 80% 的潜水率是正常的。
- 微信群管理很耗精力。如果你发现自己花太多时间在群里了，果断减少频率、缩短时间。
- 注意隐私——不要发其他人的对话记录。每个用户的对话内容都是私密的（哪怕是从 Veriloquy 的聊天记录里截的图）。

## 方向八：产品化（Web App 收费）——远期方向

### 这个方向现在别碰

我直接说了：**现在别想这个。**

你现在连 50 个活人用户都没有。现在想收费，就像刚学会炒鸡蛋就想开餐馆。

但这不代表永远不要想。这里写的是"远期方向"——当未来某一天，你可能想认真考虑"让 Veriloquy 变成一个真正的产品"时，可以参考的思路。

### 什么时候可以开始考虑

✅ **三个条件全部成熟**：
1. 有 500 个以上"真正在用"的用户（不是 Star 过 500，是每天有几百人在用）
2. 用户主动提过"我愿意付费"，不止一两个，而是很多人在问
3. 你大部分业余时间已经无法应付维护和需求的节奏

❌ **任何一个条件不满足**，就别搞收费。连想都别想。

### 如果真的到了那一天，怎么做

**方式一：Hosted 版（最简单）**

- 什么都不变，只是把 Veriloquy 部署到一个公开网址上
- 用户不需要 clone 代码、不需要配置 API Key
- 免费体验：10 次对话/天
- 付费版：无限对话 + 保存历史 + 多设备同步
- 定价：19 元/月 或 199 元/年（参考类似工具的国内定价）
- 同时保持 GitHub 版完全开源——""爱折腾自己搭，不爱折腾付费用""

**方式二：增值功能版（推荐）**

- 开源版保持 100% 免费，你甚至不需要把它变弱
- 额外付费的东西是：语音对谈、学习统计仪表盘、教材库接入大家的 UGC 等"增值"功能
- 这样的好处是：不会被骂"开源骗局"，也保护了你的真正用户

**方式三：内容订阅（适合你）**

- 你是写网文的。你可以每个月写几篇"精编教材"，只在付费版里能用
- 比如"AI 写作入门""网文结构拆解""如何用 AI 帮你构思故事大纲"
- 这跟 Veriloquy 的工具属性结合，而且你写的教材别人写不了

### 但是，现在你真正需要做的是

把以下这几件事做扎实，以后才有"产品化"的机会：
1. 让体验更好（安装流程简化、一个命令启动）
2. 积累口碑（让用过的 50 个人愿意推荐给朋友）
3. 收集反馈（知道用户真正在用 Veriloquy 学什么，学了之后有什么改变）
4. 保持开源（GitHub 版永远免费，这是你的信任资产）

### ⚠ 注意坑

- **绝对不要"先收费再开发"**。你没有用户信任基础。
- 不要在项目中插广告。这是个人项目，不是商业网站。
- 不要因为有收入了就把开源版阉割。这是最蠢的操作——你会失去所有社区信任。
- 不要过早商业化。很多人（包括大公司）在做"AI 教育"，你最大的优势就是——你不是一家公司，你是一个真诚做东西的人。保持这个状态，远比你收几万块钱月费重要。

## 优先排序：从哪个开始干？

如果时间有限，我建议的实战顺序：

### 🥇 第一优先：知乎文章（方向二）

**原因**：你是写东西的。你的最强能力是文字。2-3 天专注写一篇文章，可能是八件事里性价比最高的——投入跟产出比最高。

**立刻要做的事**：
1. 定好第一篇文章的主题（比如"我用 AI 四个夫子学了一整天"）
2. 花一下午跑一次完整的 Veriloquy 体验，截好图
3. 开始写。写完后放一天，第二天改一次再发。

### 🥈 第二优先：GitHub 优化（方向一）

这跟写文章不冲突。花一晚上整理 README 首页，做个 Demo GIF。做完这两件事，后续所有推广都会有个"门面"可以指向。

### 🥉 第三优先：V2EX/掘金 交叉发布（方向六）

知乎文章写好后，改一改就能发到掘金和 V2EX。一鱼三吃，效率极高。

### 第四：微信社群（方向七）

等你有 3-5 篇文章发出去了，有人评论"怎么用""可以试试吗"，再拉群。群不是从零建的，是从已有的互动里自然长出来的。

### 第五：B站/高校/比赛

这三件事都很耗时间。等你前面的基础打好了（有人知道了、有人试用了、反馈回来了），再考虑集中精力做一个视频或参加一个比赛。不用急。

---

## 最后的三个提醒

1. **你最大的优势不是技术，是你的文字。** 绝大多数技术项目推广不好，是因为开发者写的东西像说明书而不是故事。你会讲故事的技能，比写一万行代码都值钱。

2. **别追求完美再发布。** "等我把 V8 做完整了再推广"→ 等你做完整了，可能已经不想动了。现在这个 MVP 就可以拿出来给人看——"还在打磨中，欢迎来玩"比"这是一个成熟产品"更适合你现在的状态。

3. **做推广最需要的能力是"持续"。** 写一篇好文章、参加一次比赛，都不难。难的是半年后还在写、还在改、还在跟用户聊天。你不需要第一次就惊艳所有人，只需要一直站在这里。
