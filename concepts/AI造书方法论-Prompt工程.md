# AI 造书方法论

## 一句话

**用一份精心设计的 Prompt，让 AI 在几小时内生成一本完整、可用的教材。** 不是"AI辅助写书"——是AI作为主体写书，人做编辑和审校。

## 核心洞察

AI造书和人类写书的流程完全不同。人类写作是"想→写→改"，AI写作是"设计Prompt→执行→审校→修补"。人类作家纠结于句子，AI纠结于结构。如果把AI当人类作家用（一句一句让它写），结果会很平庸。关键是把**整本书的设计决策全部前置到Prompt里**。

## 一、Prompt 的七层前置设计

吴乐旻的《细听·古典卷》Prompt 展示了完整的设计层次：

### 第1层：定位（Positioning）
```
这本书是什么，不是什么。
```

不是简单说"这是一本古典音乐教材"，而是用 **Anti-Definition** 定义边界——

> 不是音乐学院曲式分析教科书。不是音乐史学术综述。不是"必须先学会读谱"的入门书。不是西方-中心/德奥-中心的窄路。不是"古典音乐是高雅的应该端坐欣赏"的精英主义书。

**是**："陪着你听一遍、再听一遍，然后让你想跟人聊聊这段音乐"的书。

Anti-Definition 是 Prompt 工程中最被低估的技术。说"是"什么，AI可能理解偏差。说"不是"什么，AI立刻知道边界在哪里。

### 第2层：读者（Reader）
```
谁在读，在什么场景下读，起点是什么，终点是什么。
```

- 读者画像："喜欢听但不识谱的成年业余爱好者"
- 使用场景："打开书→读1分钟预热→去YouTube搜→听完→回来读书→打开Socratopia跟伙伴聊"
- 起终点："只听过几首名曲，想知道还有什么" → "认识80首作品，能听出不同演奏家的区别，敢于走进音乐厅"

精确的读者画像不是写书时才用——它决定了每一个章节的语气、难度、轶事密度。写给"不识谱的人"和写给"音乐系学生"，同样的贝多芬第五，前者讲"四个音符怎么变成一整段音乐"，后者讲"第一主题群与第二主题群的调性关系"。

### 第3层：结构（Structure）
```
几章、几部分、每章的锚点是什么。
```

《细听·古典卷》的结构：
- 9部分 / 27章
- 80%时序（巴洛克→古典→浪漫→20世纪）+ 20%主题穿插（"如何听弦乐四重奏"）
- **每章锚定在1-3段具体音乐上**（不是讲"巴洛克音乐的特点"，而是讲"Bach Brandenburg Concerto No.3"）

这里的关键决策：**第一章不从巴赫开始（门槛太高），从贝多芬第五开始（读者已经认识）。**

结构设计中最常见的错误是按逻辑顺序排（从最早开始），而不是按心理顺序排（从读者已经知道的东西开始）。逻辑顺序对专家友好，心理顺序对学习者友好。教材要的是后者。

### 第4层：章节骨架（Chapter Skeleton）
```
每章必须是同一个结构，强制。
```

《细听·古典卷》的四段式：
1. **Before You Listen**（钩子轶事 + 最多3个注意事项 + 搜索hint）
2. **The Listening Window**（听完后的二刷提示——"这次注意第4分12秒的大提琴"）
3. **What's Happening**（结构地图 + 技术亮点）
4. **The Stories**（轶事弹药库——Socratopia伙伴聊半小时的素材）

四段式强制的原因是：**结构给AI约束，约束产生质量。** 不给结构，AI会写成散文；给了结构，AI填充内容，每章在固定骨架下做变化。质量稳定。

最关键的设计洞察在"二刷"——第二章"The Listening Window"。大多数鉴赏书让你听一遍然后告诉你"这个作品表达了什么"。这本书让你**再听一遍**，因为"第二遍才是真正听懂的起点"。这个设计不是拍脑袋——它来自播放列表时代的用户行为：人们愿意听一遍，但很少主动听第二遍。所以教材把"第二遍"内置在结构里。

### 第5层：Threading（线索编织）
```
跨章节的叙事线索，让书不只是27篇独立文章。
```

《细听·古典卷》有三条贯穿全书的线索：

**Thread A — 作曲家群像（6-8位）**：巴赫、莫扎特、贝多芬、瓦格纳、柴可夫斯基、马勒、肖斯塔科维奇。每个人不止在一章出现——每个人在不同章节累积成三维形象。读者从"德国作曲家"变成"那个聋了之后写出第九的人"。

**Thread B — Dies Irae（末日经）**：中世纪素歌旋律，被几十位作曲家引用。第16章（柏辽兹）第一次正式教读者这个旋律。之后在第20（柴可夫斯基）、第21（马勒）、第24（布里顿）、第26（约翰·亚当斯）分别出现"听——这里有 Dies Irae"。读者从"学到一段旋律"变成"获得一种超能力——在任何电影配乐里听出Dies Irae"。

**Thread C — 六个要丢掉的听乐习惯**：全书脊柱。第1章冷开场丢出六个习惯，各章在相关时刻回扣，第27章作为终章自检表收束。类比就是本书版的"六条谎言"。

Threading是区分"好教材"和"文章合集"的分水岭。没有Threading的教材是27篇博客。有Threading的教材是一本书。

### 第6层：声音DNA（Voice DNA）
```
书的声音从哪来。不是"幽默/严肃"这种空洞形容词，而是指定具体的声音来源。
```

《细听·古典卷》的Voice DNA：15个来源，每个有"为什么"的解释——

| 声音来源 | 为什么用它 |
|---------|-----------|
| Aaron Copland *What to Listen For in Music* | 友好不居高，写给业余听者的金标准 |
| Leonard Bernstein 青少年音乐会 | 热情、用比喻、表演+讲解同步 |
| Robert Greenberg Great Courses | "先听后讲"的结构原型 |
| Alex Ross *The Rest is Noise* | 20世纪叙事大师 |
| David Hurwitz 录音评论 | 尖锐的版本对比风格 |

以及要避开的声音：纯学院派曲式分析、纯八卦电视风格、"古典音乐拯救心灵"式励志。

指定Voice DNA比说"语气要友好"精确得多。AI不知道"友好"是什么边界，但知道"像Aaron Copland那样讨论音乐"是什么样的。

### 第7层：编辑规范（Editorial Spec）
```
可执行的具体格式规则，不是"注意质量"这种废话。
```

- 搜索hint约定：作品全称（英+中）、乐章/段落、约X分钟、2-3个推荐版本+一句描述、双语搜索词
- 录音推荐：每次推荐必须标注演奏家/乐团/年份/厂牌。不能写"卡拉扬有个很好的版本"——必须是"Karajan / Berlin Philharmonic 1963, Deutsche Grammophon"
- 时间戳约定：所有时间戳必须指向章节指定的"Primary Recording"
- 轶事密度要求：每章至少1个Hook Anecdote + 3-5个Extension Anecdotes + 2-3个录音史轶事 + 至少1个跨章节链接

质量从约束中产生。不指定宽度，AI写200字轶事。指定"每章至少5个轶事"，AI写出8000字。

## 二、完整造书流程

### 阶段1：构思（人 + AI对话）

不是写Prompt——是跟AI对话，让AI帮你完善想法。

吴乐旻的流程：
```
人："音乐鉴赏怎么样？"
AI："这个想法非常对路。原因一二三。建议：不要写死链接给搜索hint，四段式结构，读者切到不识谱水平..."
人："同意。章节路线呢？"
AI："80%时序+20%主题穿插。第一章不要从巴赫开始，从贝多芬第五..."
```

这个阶段不能跳过。它本质是 **Prompt的草稿**——AI帮你理清了所有设计决策，然后把决策写进Prompt。

### 阶段2：转化为Prompt（人）

把对话中的共识转成Prompt的各层。Prompt里每一条"要求"都应该来自阶段1的对话成果，不是拍脑袋编的。

**关键原则：Prompt里不写废话。** 每一条要求都需要在生成中产生可见的影响。如果不能回答"这条要求会让AI写的跟没有这条要求有什么不同"，就不写。

### 阶段3：执行生成（AI）

AI按Prompt生成全书。按《细听·古典卷》的标准：约27章，每章4000-8000字，全书约15-25万字。

### 阶段4：蓝军审校（AI自我校验）

生成后立即跑"蓝军"——用一份审校清单逐章检查：
- 轶事是否准确可查？（虚假轶事 = 教材废了）
- 录音推荐是否存在？（幻影录音 = 教材废了）
- 时间戳是否指向正确的版本？
- 各章之间的Threading线索是否连贯？
- 声音是否一致？（第3章的语调和第23章不能像两个人写的）

蓝军是**必做**的步骤。不经蓝军的AI教材不可信。吴乐旻的做法：蓝军→修→蓝军→登记→commit GitHub。

### 阶段5：登记入库

教材进入版本管理，标注生成日期、模型版本、审校轮次。这意味着教材可以迭代——模型升级后重新生成同一本教材，对比变化。

## 三、通用造书 Prompt 模板（反向工程）

从《细听·古典卷》的Prompt中提取的通用结构：

```
# [书名] — Textbook Generation Prompt

## 0. 灵魂段落（1-2段）
这本书是什么体验。不是摘要——是一个画面。读者拿起这本书时是什么样的，
读完时变成了什么样。这一段定了全部声音。

## 1. 定位
- 是：...
- 不是：...（Anti-Definition 越多越好）

## 2. 读者
- 入口画像
- 使用场景（精确到分钟）
- 出口画像（读完时能做什么、知道什么）

## 3. 结构
- 几部分 / 几章
- 章节安排原则（时序？主题？难度递进？）
- 每章的锚点（具体到"什么作品/什么概念/什么案例"）

## 4. 章节骨架（强制）
每章必须包含哪几个部分。命名、顺序、每部分的字数/功能。

## 5. Threading（至少2条贯穿线索）
- Thread A: [名称] → [如何在各章中出现]
- Thread B: [名称] → [如何串联]
- Thread C: [名称] → [开篇+回扣+收束]

## 6. Voice DNA
- 来源1: [书名/人] — 为什么
- 来源2: ...
- 避开: ...

## 7. 编辑规范
- 搜索hint/引用/数据约定
- 每章必须包含的最小内容密度
- 审校要求

## 8. 长度
约X章。每章约X字。全书约X万字。

## 附录：审校清单
- [ ] [具体检查项1]
- [ ] [具体检查项2]
```

## 四、与人类写书的比较

| 维度 | 人类写书 | AI造书 |
|------|---------|--------|
| 研究阶段 | 数周到数月 | 内化在模型里（即时的） |
| 写作速度 | 每天2000-5000字 | 几小时一本书 |
| 质量核心 | 作者的专业知识 | Prompt的设计质量 |
| 一致性 | 自然的（同一颗大脑） | 需要Voice DNA+蓝军强制 |
| 事实准确 | 依赖作者核查 | 需要蓝军逐条核查 |
| 编辑成本 | 改句子 | 改Prompt→重新生成 |
| 迭代速度 | 修改→排版→印刷→年 | 修改Prompt→跑一次→分钟 |
| 适合 | 原创思想、个人化声音 | 结构化教材、系列作品 |

AI造书不适合写《百年孤独》，但极其适合写教材。教材的特点是结构先于内容、准确性优先于风格、系列化生产——这正是AI的强项。

## 五、关键洞见

**1. 教材的结构设计是知识本身。** 章节顺序不是排版问题——它决定了读者如何构建知识体系。从已知到未知、从简单到复杂、从具体到抽象——这些不是"教学技巧"，是认知规律。一个好的教材Prompt做的最重要的事不是定义内容，是定义认知路径。

**2. Anti-Definition 比 Definition 更有用。** 说"这本书是什么"可能被误解；说"这本书不是那种XX书"AI立刻知道边界。这个原则适用于所有Prompt工程。

**3. Threading 决定教材是"一本书"还是"一堆文章"。** 没有贯穿线索的教材读完后读者脑子里的知识是离散的；有Threading的教材，每条线索都是一根绳子——读者可以抓住一端把全部知识拉起来。

**4. 蓝军不能省略。** AI比人类作家更擅长生成，也更擅长编造。一个虚构的轶事、一个不存在的录音版本，足以毁掉整本教材的信誉。AI造书的质量 = Prompt质量 × 蓝军质量。

**5. 造书的瓶颈不是AI，是人的好问题。** 任何有清晰领域的教材都可以用这个方法在主食化——古典音乐、爵士乐、中国民乐、电影史、建筑史、烹饪科学、经济学入门、编程语言。瓶颈不是AI能不能写，而是你能不能提出那个"值得写一本书"的问题。
