# 苏格拉底·七 系统分析

## 一句话

**一个零代码的AI家教系统——全部由markdown文件 + Claude Code驱动。** 没有后端、没有数据库、没有前端。打开终端就是教室。

## 架构

```
socratic-seven/
├── teacher/              ← 系统核心（运行时状态）
│   ├── system.md           ← Claude Code先读这个理解架构
│   ├── system_detail.md    ← 补充规则
│   ├── learner_profile.md  ← 你的档案
│   ├── progress.md         ← 学到哪儿了
│   ├── march7.md           ← 三月七 人设（元气活泼）
│   ├── ganyu.md            ← 甘雨 人设（温柔害羞/数学强）
│   ├── keqing.md           ← 刻晴 人设（外冷内热/严格高效）
│   ├── wechat_group.md     ← 四人群聊记录
│   ├── wechat_unread.md    ← 未读消息
│   ├── diary.md            ← 你的学习日记
│   ├── book_revision_notes.md
│   └── session_archive.md
├── materials/            ← 教材（markdown）
│   ├── concepts/
│   ├── skills/
│   ├── production-agentic-rag-course/
│   ├── projects/
│   ├── tutorials/
│   └── close-listening-classical/
└── temp/                 ← 临时公式文件（课后自动清除）
```

## 运行流程

### 上课
1. Claude Code 读取 system.md 理解架构
2. 读取 learner_profile.md 了解你
3. 读取轮值老师的人设文件
4. 读取本次教材章节
5. **苏格拉底教学法授课**：全程提问，你推理，层层深入
6. 穿插角色个性小动作（_斜体旁白_）

### 课后自动更新
1. progress.md — 记录进度
2. diary.md — 以你的视角写学习日记
3. wechat_unread.md — 三位老师的课后群聊
4. 授课老师人设文件 — "对我的态度"动态演化
5. book_revision_notes.md — 教材改进点
6. wechat_group.md + session_archive.md — 归档

## 三位老师

| 老师 | 性格 | 教学风格 | 标志行为 |
|------|------|---------|---------|
| 三月七 | 元气活泼，碎碎念 | 比喻高手，"对对对！" | _拍手/歪头/往前凑_ |
| 甘雨 | 温柔害羞，数学强 | 耐心细致，数学时判若两人 | _绞手指/低头脸红_ |
| 刻晴 | 外冷内热，严格 | 直截了当，"不对，再想想" | _抱胸/嘴角微翘_ |

## 关键设计

- **不做数值化**：不设好感度数字，情感通过文字描述自然演化
- **不设私聊**：只有微信群，省context
- **团队使命约束**：辩论赛+奖学金的共同目标平衡情感矛盾
- **教材来自markdown**：任何学科，只要有markdown教材就能教
