# 知乎回答：我让两个 AI Agent 交叉审计了一个学习系统，发现它只用了 2/8 的功能

> 问题：「你在 AI Agent 项目开发中踩过哪些坑？」

---

刚经历了一次让我对整个 Agent 系统设计产生全新理解的事。

我一直在用自己搭的一个 AI 家教系统——叫 **Socratic Verse**——学 AI Agent 课程。系统设计得挺好：8 种主动学习方法（从苏格拉底追问到费曼教学到间隔重复），3 位 AI 教师轮值教学，全部由 markdown 文件驱动。

上完 6 节课，5 篇教材全部标记 ✅。看起来一切正常。

直到我让 Hermes Agent 和 Claude Code **各自独立分析同一个系统**，然后交叉验证结果。

结果发现——

**这系统宣传"8 种学习方法"，实际执行过的只有 2 种。**

V1 苏格拉底用了 5 次，V7 对子互教用了 1 次。剩下的 V2 费曼、V3 Anki、V4 骰子、V5 合书复述、V6 交错魔鬼、V8 自适应——全写在 `prompts/system-prompts.md` 里，但**从来没有被任何机制触发过**。

用户必须手动说"请用 V5 模式"——但在沉浸式学习过程中，没人会记得手动切换。

---

## 双 Agent 交叉验证发现的 5 个坑

### 坑 1：扫读 ≠ 学习（系统误判了学习状态）

做教材关联图谱分析时，两个 Agent 都把教材通读了一遍。结果系统把这次"分析扫读"标记为"已学习"——在进度表里标了 ✅。

分析模式和学习模式共享了同一个进度标记。这是任何 Agent 系统都可能犯的错误：两种语义上截然不同的行为，被同一个状态机记录。

**修复**：在 `system.md` 新增加章节，用关键词识别区分两种模式——"分析/扫读"识别为分析模式（不更新学习进度），"教我/上课"识别为学习模式。并在 agent-course-learning skill 中明确："分析过的教材仍需正常上课。"

---

### 坑 2：V2-V8 大面积"幽灵功能"

宣传有 8 种方法，但系统没有任何"课后自动触发其他方法"的机制。V3 Anki 需要题库——题库从未建立。V4 骰子需要题库——同样没建。

最致命的是，系统没有任何"学习方法调度表"。什么时候该用 V5 合书复述？什么时候该用 V2 费曼？全是黑盒。

**修复**：建立了完整的调度表（`learning_schedule.md`）。现在一节课的生命周期是：

```
V1 上课 → 课后 1h 自动 V5 合书复述 → 24h 自动 V2 费曼 → 72h 自动 V7 对子互教 → 7d 入 V3 Anki → 14d 入 V4 骰子
```

同时生成了 25 道 Anki 推理题 + 20 道骰子题，设定了每日 V3（08:00）和每周 V4（周日 20:00）cron。

---

### 坑 3：教师轮值的隐藏 Bug

系统有三个老师：三月七（元气活泼）、甘雨（温柔数学）、刻晴（严格高效）。旧规则说"轮值顺序是三月七→甘雨→刻晴"。

但实际执行时，"驾驭工程"这篇教材讲的是六层 Harness 架构——三月七和甘雨都不擅长。结果三月七硬着头皮教了一轮，效果不好，刻晴又重讲一遍。浪费一节课。

规则只说了"顺序"，没说"学科偏好不匹配怎么办"。

**修复**：学科偏好升级为软约束——偏差 ≤1 按偏好，偏差 ≥2 按轮值。驾驭工程固定由刻晴教。同时新增了第四位老师芙宁娜（复旦哲学系，"元认知/思辨"方向），填补了"概念辨析"教材没有匹配老师的问题。

---

### 坑 4：教材编号反认知

三篇权威参考文献（Anthropic 原文、OpenAI SDK、LilianWeng 学术综述）的 order 编号是 1-3，排在所有核心教材前面。按编号学，结果是先读英文权威文献再读原创教材——完全反了。

参考文献应该是"学完核心概念之后的反刍材料"，而不是"前置挡路石"。

**修复**：8 篇教材 frontmatter 全改了——`order: N` → `core_chain: 1-5`（核心教材）和 `reference: R1-R3`（参考文献）。编号不再误导学习路径。

---

### 坑 5：复习机制完全缺失（最严重的一个）

5 篇核心教材全部标 ✅ 之后，系统没有任何后续复习计划。2026-06-16 学的"理解Agent"到 06-22 已经过了 6 天——按艾宾浩斯遗忘曲线，已遗忘约 50%。

`progress.md` 只有"学了什么"，没有"什么时候该复习什么"。多方法体系的设计里有 V3 Anki（间隔重复），但它从未启用。

**修复**：progress.md 新增"首次学习/最后复习/下次复习/复习次数/Anki"四字段。每节 V1 课后自动生成 5 道 Anki 推理题，加入每日推送池。

---

## 双 Agent 协作的核心价值

两个 Agent 独立分析同一系统后交叉验证——共识部分直接执行，差异部分反而产生了新洞察。

**Claude Code 独有发现**：V2 费曼是 V1 苏格拉底的"镜像"——一个是老师追问学生，一个是学生教老师。之前只用前者，忽略了后者的互补价值。

**Hermes 独有发现**：关联图谱中识别了 3 处跨教材概念矛盾（如 Agent vs Workflow 分类法在 Anthropic 和理解Agent 中定义不一致）。

这次改进的核心不是"修了多少 bug"——是验证了一套可复用的双 Agent 协作框架：**分工定义 → 并行执行 → 合并对比 → 方案选择 → 执行验证**。任何一个 Agent 单干都能完成 70%，但剩下的 30%——那些唯独来自另一份分析的新视角——才是真正的价值。
