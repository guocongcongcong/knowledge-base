# 知识库压缩工具调研

调研背景：为 Claude + Obsidian 知识库寻找能大量压缩记忆的工具。以下工具代表了从结构化索引到后台自动压缩的几种不同路线。

---

## 工具速览

### @ctxr/skill-llm-wiki — 结构化索引

- **路线**：将文件夹结构化为确定性、Token高效的检索库
- **压缩核心**：成本与语料库大小解耦。约 **90% 的检索无需调用 LLM**，仅用本地 TF-IDF + MiniLM 完成
- **Token 成本**：与查询模糊度相关，而非库大小
- **适用场景**：愿意对知识库做结构化梳理

### Enzyme — 语义压缩层

- **路线**：利用 Obsidian 原生结构作为上下文信号
- **压缩核心**：读取标签、链接、文件夹结构，为 Claude Code 提供跨会话持久记忆
- **效果**：相比关键词搜索，实现 **2-12 倍的 Token 压缩**
- **适用场景**：保留现有 Obsidian 工作流，利用已有结构增强 AI 记忆

### apsolut-cortex — 后台自动压缩

- **路线**：全自动记录 + 压缩
- **压缩核心**：自动记录 Claude Code 会话，会话结束时通过 Claude Haiku 将"观察"压缩为结构化记忆
- **适用场景**：完全不想改变现有习惯，只需全自动记忆增强器

### SimpleMem — 论文级语义无损压缩

- **路线**：基于学术研究的语义无损压缩框架
- **压缩核心**：三阶段流水线
  1. 语义结构化压缩
  2. 递归记忆整合
  3. 自适应查询感知检索
- **效果**：Token 消耗最多降低 **30 倍**，平均 F1 分数提升 **26.4%**
- **适用场景**：追求理论最优压缩效果

### File Compressor — 物理存储压缩

- **路线**：物理文件压缩
- **压缩核心**：引入 `.ctxt` 压缩文件格式，可压缩文件大小高达 **70%**
- **注意**：与记忆压缩需求不同，属于存储优化方向
